# Features Workspace

This directory stores product intent, design rationale, execution plans, and generated evidence for the Harness project.

## Structure

- `product-specs/`: versioned requirements and expected behavior.
- `design-docs/`: architecture rationale and engineering principles.
- `exec-plans/active/`: living implementation plans.
- `exec-plans/completed/`: historical plans.
- `generated/`: generated reports and derived artifacts.
- `tech-debt-tracker.md`: deferred work backlog.

## Usage

1. Start from product specs (`what`).
2. Update design docs for architecture decisions (`why/how`).
3. Create/maintain ExecPlans for non-trivial implementation (`execution`).
4. Store generated evidence in `generated/`.
